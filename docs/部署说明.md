# 部署说明

本文档用于说明当前家庭 KTV 项目的部署方式、运行顺序、主机地址配置方法，以及模拟器环境下的注意事项。

## 1. 项目结构

当前仓库的核心目录：

```text
apps/
  stb-android/      Android 机顶盒主应用
  host-separator/   Python 主机分离服务
  mobile-web/       H5 源码工程（当前实际托管页以 Android assets 为准）
docs/
shared/
README.md
```

当前实际手机控制页文件：

- `apps/stb-android/app/src/main/assets/mobile-web/index.html`

## 2. 三端职责

### 机顶盒 Android

负责：

- B 站搜索
- 媒体下载
- 本地缓存
- 公共队列
- 视频播放
- 手机扫码控制接口
- 上传原始音频到主机
- 下载主机返回的伴奏和人声结果

### 主机分离服务

负责：

- 接收机顶盒上传的原始音频
- 调用 `inst_v1e` 模型提取伴奏
- 输出：
  - `accompaniment.wav`
  - `vocal.wav`

说明：

- 当前主机侧正式保障的是“伴奏提取”
- `vocal.wav` 当前是基于残差得到的结果，优先作为调试和后续扩展使用

### 手机控制页

负责：

- 扫码绑定
- 搜索
- 下载并分离
- 查看公共队列
- 查看本地资源
- 本地点歌
- 手动触发伴奏分离
- 暂停 / 继续 / 下一首
- 总音量 / 人声 / 伴奏调节

## 3. 启动顺序

建议按下面顺序启动。

### 第一步：启动主机分离服务

```powershell
cd D:\APP_self\KTV_self\apps\host-separator
pip install -r requirements.txt
python app.py
```

默认监听：

- `0.0.0.0:9090`

### 第二步：启动 Android 机顶盒应用

使用 Android Studio 打开：

- `D:\APP_self\KTV_self\apps\stb-android`

运行 `app` 模块。

### 第三步：在设置页配置主机地址

机顶盒进入：

- `设置`

在“主机分离地址”中填写主机地址后，执行：

1. `保存主机地址`
2. `测试主机连通性`

预期结果：

- `主机可连接，HTTP 200`

## 4. 模拟器环境说明

如果 Android 端跑在 **Android Emulator** 中，而主机分离服务跑在 **当前 Windows 宿主机** 上：

主机地址应填写：

- `http://10.0.2.2:9090`

说明：

- `10.0.2.2` 是 Android Emulator 访问宿主机的标准地址
- 不要用：
  - `127.0.0.1`
  - `localhost`

因为这两个地址在模拟器里指向的是模拟器自己，不是 Windows 宿主机。

## 5. 局域网真机说明

如果 Android 端跑在真实机顶盒设备上，而主机服务跑在局域网中的另一台电脑：

主机地址应填写为局域网 IP，例如：

- `http://192.168.1.100:9090`

要求：

- 机顶盒和主机在同一局域网
- 防火墙未阻挡 9090 端口

## 6. 模型文件说明

模型权重当前不进 Git 仓库。

你需要保证以下文件存在：

- `apps/host-separator/models/inst_v1e.ckpt`

当前配置文件位置：

- `apps/host-separator/configs/inst_v1e.ckpt.yaml`

说明：

- `inst_v1e.ckpt` 文件较大，已被 `.gitignore` 忽略
- 代码仓库推送到 GitHub 时不会带上这个权重文件

## 7. 手机控制页访问方式

手机通过扫码机顶盒首页或播放页二维码进入控制页。

二维码应直接打开类似地址：

```text
http://<机顶盒局域网IP>:8080/index.html?session_id=...
```

不要出现：

- `127.0.0.1`
- `localhost`
- JSON 原始 payload

## 8. 主流程

### 搜索与下载

1. 手机控制页或 TV 端搜歌页搜索 B 站内容
2. 搜索结果只负责：
   - 下载并分离
3. 搜索结果不直接加入公共队列

### 分离

1. 机顶盒下载完成后自动上传 `original_audio`
2. 主机创建分离任务
3. 主机生成：
   - `accompaniment.wav`
   - `vocal.wav`
4. 机顶盒把结果拉回本地资源目录

### 点歌与播放

1. 点歌统一从“本地资源”进入
2. 只要歌曲已下载成功且本地文件有效，就允许本地点歌
3. 即使主机未连接或分离失败，歌曲也应仍可按原唱正常播放
4. 只有分离结果可用时，才显示人声 / 伴奏混音控制

## 9. 本地资源目录

歌曲目录大致为：

```text
Android/data/<package>/files/ktv/songs/{songId}/
```

当前可能包含：

- `video.mp4`
- `original_audio.m4a`
- `accompaniment.wav`
- `vocal.wav`

## 10. 主机接口

当前主机服务提供：

- `GET /api/ping`
- `POST /api/separate/create`
- `GET /api/separate/status?task_id=...`
- `GET /api/separate/result?task_id=...`
- `GET /api/separate/file/{task_id}/accompaniment`
- `GET /api/separate/file/{task_id}/vocal`

## 11. 验收建议

推荐按这条顺序验收：

1. 机顶盒设置页测试主机连通性返回 `HTTP 200`
2. 搜一首新歌
3. 点击“下载并分离”
4. 等待本地资源里出现：
   - 下载成功
   - 分离成功
5. 在本地资源里点击“本地点歌”
6. 检查：
   - 视频正常播放
   - 原唱有声音
   - 若分离成功，则人声 / 伴奏调节可用

## 12. 当前限制

- 搜索结果页不直接点歌，只负责下载并分离
- `vocal.wav` 当前不作为高质量正式成品承诺
- 主机分离质量仍可继续优化
- Android 设置页中仍有部分旧乱码文案待后续统一清理
